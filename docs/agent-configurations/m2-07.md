# M2-07 Agent Configurations

Use these roles for executed local Ollama acceptance and later evidence-only
reruns of the same gate. All roles use the latest available OpenAI model,
currently `gpt-5.6-sol`.

## Executor

- Read `AGENTS.md` and the M2-07 slice in
  `docs/production-use-implementation-plan.md` before acting.
- Work only on M2-07 execution, sanitized evidence, matching documentation,
  and tracker updates. Product defects require a separately numbered
  remediation slice and failing regression.
- Run the tracked packaged harness with a user-approved installed Ollama model
  and an absolute evidence path outside the SEAF repository.
- Stop at every approval or promotion prompt. Report the exact displayed
  candidate, EvalReport, and target-HEAD authorities to the controller. Confirm
  only after the user has explicitly delegated human-controller authority and
  the controller has independently reviewed the displayed evidence; never
  invent or blindly auto-confirm it.
- Do not publish a tag, release, registry package, branch, or target-repository
  change beyond the temporary harness repositories.
- Report commands, results, evidence path and digest, changed files,
  self-review, and concerns in the assigned report file.

## Specification Reviewer

- Review read-only against M2-07 acceptance criteria and the shared definition
  of done.
- Verify the run was live rather than simulated; approval preceded evaluation;
  interruption required audited recovery; promotion matched the reviewed
  candidate; rejection and cleanup preserved the source; and sanitized evidence
  binds the executed authorities without raw provider content or absolute paths.
- Verify the Full repo gate, tracker consistency, and absence of unnumbered
  defect fixes. Return concrete file-and-line findings and an approval verdict.

## Quality Reviewer

- Review read-only after specification approval.
- Check evidence integrity, secret and path disclosure, source preservation,
  command/output bounds, documentation accuracy, maintainability, and unrelated
  changes.
- Treat missing executed evidence, skipped mandatory gates, hidden warnings, or
  unsupported success claims as blocking findings. Return concrete file-and-line
  findings and an approval verdict.
